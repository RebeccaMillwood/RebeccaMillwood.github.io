HTML & CSS

My task was to create a portfolio site to showcase my work; and this site is the outcome of this project.

To meet the requirements of this project, my site needed to include a profile picture, biography, a functional contact form, sections dedicated to projects I have worked on and links to external sites (I have links to my GitHub and LinkedIn pages).

In addition to the content requirements, the technical requirements were to have at least 2 pages, be deployed on GitHub, be responsive and use semantic HTML.

I also included the following additional features: a different font colour when hovering over links, and a a simple JavaScript function to display a hidden message on my home page.

Python

For this project, I was provided with two json files containing weather data; one contained a 5 day forecast, the other historical data for a 24 hour period. My task was to convert the data into something useful.

The requirements for this project were to write a Python script that produced: a summary of the forecast, a summary of each day and needed to meet specific output formatting, and pass tests that checked functions in my code (specifically the functions to convert Fahrenheit to Celsius, calculate the Mean and the overall output function).

The second part of this project required two graphs to be produced, showing the minimum and maximum temperatures for each day, and the minimum, minimum ‘real feel’ and minimum ‘real feel shade’ temperatures. I used Plotly to produce two line graphs with the expected output.

Additional points to note were that all dates and times in the output needed to be in a human readable format, all temperatures needed to be converted from Fahrenheit to Celsius and displayed accordingly, all graphs needed a main title, titles for the X & Y axis and a legend.

Django Rest Framework (DRF)

The Django Rest Framework (DRF) project was Part A of a wider Crowdfunding project - where I created my own Crowdfunding site.

A requirement for the wider project included the completion of an MVP plan - the minimum viable product. This needed to include the title of the website, the purpose and target audience, an API specification, a database schema, wireframes, colour scheme and fonts to be used.

For Part A, I built the backend database and set up API end points using DRF, and tested them using Insomnia.

The GET method was set up to allow all projects to be viewed, as well as individual projects, to view all pledges and individual pledges, and to view user data.

The POST method was set up to allow a new user to register themselves on the site, to create a new project, to be able to pledge to a project, and to obtain a token for authorisation.

The PUT method was set up to allow a user to edit their own details, edit their project or allow pledges to be edited.

The DELETE method was set up to allow a user to delete their project, pledge and their own user account.

React

The React project formed Part B of the wider Crowdfunding project.

React was used to create the actual website for my Crowdfunding site; DogShare

Functional pages and forms were set up to link to the API endpoints created in DRF and the site was deployed via Heroku.

The site includes the ability to register as a new user, login, view projects, create projects and edit projects. The pledge option is not currently functional. There is also a custom '404 page not found’ page instead of the default error page when a user tries to access a link that does not exist, or if they try to edit a project they are not authorised to do so.

The functionality of the pages and forms was set up to match the API endpoints and database schema that was prepared using DRF in Part A.
